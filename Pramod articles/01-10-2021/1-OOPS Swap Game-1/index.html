Understanding problem:

Dear reader, let us try to play a game by trying to swap two objects of a class. Have a look at the code given below:

import java.util.*; public class Main { public static class Person { int age; String name; void saysHi() { System.out.println(name + "[" + age + "] says hi"); } } public static void main(String[] args) throws Exception { Person p1 = new Person(); p1.age=10; p1.name="A"; Person p2 = new Person(); p2.age=20; p2.name="B"; p1.saysHi(); p2.saysHi(); swap1(p1,p2); p1.saysHi(); p2.saysHi(); } public static void swap1(Person psn1, Person psn2) { Person temp = psn1; psn1 = psn2; psn2 = temp; } }
java; true";

You have to try to predict the output of the code after swapping the objects using the swap1() method. We have used the saysHi() method to print the data of the objects before and after swapping them.

Approach :

So, dear reader, did you get the answer? Let us not waste any time and give you the answer first. The output for the above code is shown below:

So, dear reader, did you get it right? What does this output show you? This shows that the objects were not swapped actually. So, let us try to understand the reason behind this. Have a look at the image shown below:

So, when we enter the main() function, we have made two objects of the Person class. Their references are made in the stack whereas the objects are made inside the heap as shown above. Now, we display the details of these objects by calling the saysHi() function for both of them that we are not showing in this diagram.
Let us move to the call to the swap1() method. When the swap1() method is called, the method argument psn1 catches the p1 object i.e. psn1 also refers to the p1 object. Similar will be the case with psn2 as shown in the figure below:

Also, to keep the diagram simple, let us remove the lines pointing to the objects from the references as it is clear from the stack that which reference stores the address to which object. One more thing, we also have a Person temp inside the method swap1(). This is not an object (as we have not used new) rather a reference.
The first line of the swap function says temp = psn1. This means that temp is also a reference to the object with address 4k. This is shown in the image below:

So, temp = 4k as per the first line of the swap program. Now, next we say that psn1 = psn2. This means that psn1 will now hold the address to the object whose address is being held by psn2. Hence psn1 = 5k.

Now, we will move to the last statement. Here, psn2 = temp. This means that psn2 will now hold the address to the object whose address is present in temp i.e. 4k. Hence, psn2 = 4k.
So, psn1 holds the reference to the second object whereas the psn2 holds the reference to the first object as shown in the figure below:

Now, since we have completed the work inside the swap1() method, its variables will be deleted from the stack as shown in the figure.

So, if you notice, there is no change inside the main method. Hence, the swapping did take place but only inside the swap1() method and not the main method. Hence, we see the same output before and after swapping.

Approach:

This is going to be a very basic discussion around Classes and Objects. And it is always better to understand a new concept practically. So let's define a ‘class’ ‘Person’.

Now this class can have data members which can help us to define the characteristics of a person like a person’s name, age, etc.

In addition to this, a class can also have ‘functions’, like a function ‘sayhi’, using which a line can be printed, “abc [abc’s age] says hi”.

Now using this ‘person’ class we can create objects. Let's say we create an object ‘p1’. After that we can set the age of the object ‘p1’ to ‘10’ and it’s name to ‘A’. Finally we can call p1.saysHi() which will print, “A [10] says hi”.

Similarly we can create another object ‘p2’. And set its name and age as well. Then we can it’s saysHi() function too.

Let’s try to code this.

import java.util.*; public class Main { //class Person public static class Person { //data members int age; String name; //function void saysHi() { System.println(name + "[" + age + "] says hi"); } } public static void main(String[] args) { //object p1 Person p1 = new Person(); p1.age = 10; p1.name = "A"; p1.saysHi(); //object p2 Person p2 = new Person(); p2.age = 20; p2.name = "B"; p2.saysHi(); } }
java; true";

“Encapsulation is putting together data members and functions in an entity called ‘class’.”

Let’s take a look at memory and analyse, how does that work and what changes happen in the memory stack and heap.

First of all on creating object p1, a new reference was created in the memory stack which stores some address say 4k and because p1 was a new object therefore some space was allocated to it at address 4k which has space for storing p1’s age and name. On setting p1.age to 10, the age of p1 is set to 10 in the memory at address 4k. Similarly on setting p1.name to ‘A’, the name of p1 is set to ‘A’ in the memory. And on calling p1.saysHi(), “A [10] says hi” will be printed at address 4k.

Similarly work is done in memory corresponding to object p2.

At this point it is also important to understand that ‘object’ can be a misleading term at times while we understand the memory allocation of a new object. Instead we can use terms, reference and instance.
So p1 and p2 in the memory stack are reference whereas memory allocated at 4k and 5k are instances.
Now let’s say we create one more object p3 and this time instead of defining a new person, initialize it to p1.
So in the memory stack, the address that will get stored corresponding to p3 would be the same as that of p1 which is 4k and here no new instance will be created instead the p3 will point towards the same instance as that of p1 which is stored at 4k in the memory

Now, if we set the age of p3 then what do you think would happen? See, if at this point you set the age of p3 to let say 30 then the instance created at 4k will get updated. And the age at 4k will change from 10 to 30.

If we now call p3.saysHi() then the output will be, “A [30] says hi”. In fact the output for p1.saysHi() will also be the same.

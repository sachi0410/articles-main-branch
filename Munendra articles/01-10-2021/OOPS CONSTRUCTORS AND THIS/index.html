OOPS CONSTRUCTORS AND THIS

Hey coder. In this article we are going to discuss the concept of OOPS further and study constructors and the keyword this.


Important Links : Problem link

We have previously been learning OOPS using a Person class with data members age and name and a function saysHi().

Approach:

We have updated that code. Have a look:

import java.util.*; public class Main{ public static class Person{ int age; String name; void saysHi(){ System.out.println(name+"["+age+"] says Hi"); } Person(){ //1 Default constructor } Person(int age,String name){ //2 Parameterized constructor this.age=age; this.name=name; } } public static void main(String[]args){ Person p1=new Person(); //3 p1.age=10; p1.name="A"; p1.saysHi(); Person p2=new Person(20,"B"); //4 p2.saysHi(); } }
java; true";

For better understanding, watch part of the lecture video.

  • In this code, object p1 of class Person is the same as before. In //3 since we written “new Person()”, it means some function is called. But what is this function?
CONSTRUCTORS
  • A Constructor is that function which gets automatically called whenever an object of class is made using new.
    The name of this constructor is the same as the name of the class (here Person).
  • There is a constructor present in every class by default which is shown by //1.
    This means even if we forget to provide a constructor to some class then Java will provide a default constructor to it.
    This constructor has no parameters. Hence for object p1, a default constructor is called because no parameters are passed to it.
  • We can also provide a parameterized constructor to a class. This is shown by //2.
    In our example, the data members, age and name are given to it as the parameters. For the object p2 of class Person, the parameterized constructor will be called because the parameters passed are 20 and “B”.
this KEYWORD
  • Now you might be wondering what is this “this” keyword that is written under the parameterized function. We’ll understand it through a memory map.
  • When //3 is run, a new object p1 is formed and some space is allocated to it. The size of this space is the total size used by its data members, age and name and by default they are initialized to 0 and null respectively. Next, when the codes p1.age=10 and p1.name=“A” are run, the values of age and name get updated accordingly.
  • Now, for the object p2 of class Person, space will be allocated to it and through parameterized call, the data members store the values 20 and “B” passed to them. In such a situation, a keyword “this” automatically stores the address of the allocated space which is 5k here.
  • So, in the parameterized constructor, this.age and this.name denote the data members of the class as shown below.
  • And the age and name to the right whose value is being assigned to this.age and this.name denote the parameters passed to the constructor (20 and “B” here).
  • Hence, to summarise whenever an object of a class is made the following steps take place:
  1. Space allocation.
  2. Parsing
  3. Constructor firing
